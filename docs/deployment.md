# Deployment Guide

## Local Development Setup

### Prerequisites
- Node.js 20+
- Python 3.11+ (for Pipecat bot, optional)
- PostgreSQL (or Neon serverless account)
- npm

### 1. Install Dependencies

```bash
npm install
```

### 2. Environment Variables

Create a `.env` file at the project root with all required variables:

```env
# Database (Neon serverless PostgreSQL)
DATABASE_URL=postgresql://user:pass@host/dbname?sslmode=require
# OR individual vars:
# PGHOST=...
# PGUSER=...
# PGPASSWORD=...
# PGDATABASE=...
# PGPORT=5432

# AI Services
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...

# Vector Database
PINECONE_API_KEY=...

# Video Avatars
HEYGEN_API_KEY=...
HEYGEN_CREDIT_LIMIT=1000

# Voice
ELEVENLABS_API_KEY=...
ELEVENLABS_AGENT_ID=agent_...
DEEPGRAM_API_KEY=...

# Memory
MEM0_API_KEY=...

# Auth
SESSION_SECRET=your-random-secret
ADMIN_SECRET=your-admin-secret

# WebRTC
LIVEKIT_API_KEY=...
LIVEKIT_API_SECRET=...
LIVEKIT_URL=wss://your-livekit-url

# Google Services
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_SEARCH_API_KEY=...        # Optional
GOOGLE_SEARCH_ENGINE_ID=...      # Optional

# Billing
MEMBERSTACK_SECRET_KEY=...

# Email
RESEND_API_KEY=...

# Object Storage (Replit-specific, may need alternative for local)
REPLIT_OBJECT_STORAGE_BUCKET_ID=...

# Performance
NODE_OPTIONS=--max-old-space-size=4096
PORT=5000
```

### 3. Database Setup

```bash
# Push schema to database
npm run db:push

# Verify database connection
node check-and-setup-db.js
```

### 4. Start Development Server

```bash
npm run dev
```

This starts the Express server with tsx (TypeScript execution) on port 5000. Vite dev middleware serves the React client with HMR.

**Dev mode features**:
- Mock user authentication (bypasses Replit OIDC)
- Hot module replacement for frontend
- Detailed request logging

### 5. Access the App

- **App**: http://localhost:5000
- **Admin**: http://localhost:5000/admin (use admin secret)
- **Metrics**: http://localhost:5000/metrics

## Build Process

```bash
npm run build
```

This runs two steps:

1. **Vite build** — Bundles React frontend
   - Input: `client/` directory
   - Output: `dist/public/` (static assets)
   - Includes code splitting, lazy loading, tree shaking

2. **esbuild** — Bundles Express server
   - Input: `server/index.ts`
   - Output: `dist/index.js` (single ESM file)
   - External packages (node_modules not bundled)
   - Platform: Node.js

## Production Deployment

### Run Production Build

```bash
NODE_ENV=production node dist/index.js
```

The server:
- Serves static client files from `dist/public/`
- Listens on `0.0.0.0:5000` (configurable via `PORT`)
- Runs startup tasks (avatar seeding, plan init, video recovery)

### Current Deployment: Replit (GCE)

The project is currently deployed on Replit with Google Compute Engine target:
- `.replit` config defines build/start commands
- Node 20 + Python 3.11 + PostgreSQL 16 modules
- System packages: ffmpeg, imagemagick, unzip
- Object storage via Replit sidecar (port 1106)

### Alternative Hosting

For deploying outside Replit:

**What needs to change**:

1. **Object Storage**: Replace Replit Object Storage (`server/objectStorage.ts`) with a direct GCS or S3 setup. The current implementation uses a Replit sidecar for credential exchange at `http://127.0.0.1:1106`.

2. **Authentication**: Replace Replit OIDC (`server/replitAuth.ts`) with another auth provider or rely solely on Memberstack + admin secret.

3. **Environment**: Ensure all env vars are set. The `.replit` file sets some shared vars (`HEYGEN_CREDIT_LIMIT`, `NODE_OPTIONS`, `ELEVENLABS_AGENT_ID`) that need to be in your env.

4. **Database**: Neon serverless works from anywhere (uses WebSocket transport). No changes needed.

5. **System dependencies**: Install ffmpeg, imagemagick, unzip if using document/video processing.

**Deployment options**:
- **Docker**: Create a Dockerfile with Node 20, install system deps, copy dist/, run `node dist/index.js`
- **Railway/Render**: Set build command to `npm run build`, start command to `npm run start`
- **VPS**: Direct node execution with pm2 or systemd

### Docker Example

```dockerfile
FROM node:20-slim

RUN apt-get update && apt-get install -y ffmpeg imagemagick unzip && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY package*.json ./
RUN npm ci --production
COPY dist/ ./dist/

ENV NODE_ENV=production
ENV PORT=5000
EXPOSE 5000

CMD ["node", "dist/index.js"]
```

## Database Management

### Push Schema Changes

```bash
npm run db:push
```

Uses drizzle-kit to push schema from `shared/schema.ts` to the database. This is a non-destructive operation — it adds new tables/columns but won't drop existing ones.

### Migration Files

SQL migrations are in `migrations/`. These are generated by drizzle-kit but the project primarily uses `db:push` for schema updates.

### Type Checking

```bash
npm run check
```

Runs `tsc` to check TypeScript types across client, server, and shared code.

## Port Configuration

| Port | Service |
|------|---------|
| 5000 | Express server (HTTP + WebSocket) |
| 80 | External port (Replit mapping) |
| 1106 | Replit object storage sidecar |

The server binds to `0.0.0.0` with `reusePort: true`.
